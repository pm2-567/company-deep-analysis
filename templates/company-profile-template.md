## 0. 业务模式标签判定规则(必读,先做这步再填基本信息)

**核心原则**:用**公司行业归属标签**作为业务模式标签的主判定依据,**不**用公司名/简介里的关键词匹配。

### 0.1 主判定:申万一级行业映射(从 f10.公司概况 读取)

**数据源**:`/tmp/{代码}_data.json` → `f10.公司概况.所属行业` 字段(东财接口,已结构化)。

| 申万一级行业 | 业务模式标签 | 框架适配 |
|---|---|---|
| **传媒 / 计算机 / 通信 / 电子** | 互联网/平台/TOC SaaS/TOC+TOB 双边(按主业占比二选) | 框架 B(平台型五力)+ 框架 B(平台型护城河) |
| **银行 / 非银金融** | 金融 | 框架 A + 框架 D(金融护城河) |
| **汽车 / 机械设备 / 电气设备 / 国防军工** | 工业品/制造 | 框架 A + 框架 C(制造业护城河) |
| **食品饮料 / 纺织服饰 / 家用电器 / 轻工制造 / 美容护理 / 商贸零售 / 社会服务** | 消费品 | 框架 A + 框架 A(消费品牌护城河) |
| **医药生物** | 医药 | 框架 A + 框架 E(医药护城河) |
| **房地产 / 建筑装饰 / 建筑材料** | 地产/建筑 | 框架 A + 框架 F(地产护城河) |
| **基础化工 / 钢铁 / 有色金属 / 煤炭 / 石油石化 / 公用事业** | 强周期/资源 | 框架 A + 框架 C(制造业护城河) |
| **农林牧渔 / 交通运输 / 环保** | 其他 | 框架 A + 经典 6 类护城河 |

> 注:申万一级分类每年 1 次微调,以数据采集时的最新分类为准。

### 0.2 复核:主业占比判定(避免"概念误归类")

**问题**:很多公司"申万分类 = 计算机"但主业 80% 是工业品销售(如某些"工业互联网"公司)→ 不能用框架 B。

**复核规则**:
1. 读取 `f10.主营构成` 字段,按**营收占比**排序,取第一大业务。
2. 若第一大业务占比 > 70% → **强制用第一大业务的行业归属**覆盖 0.1 标签。
3. 若第一大业务占比 50%-70% → **保留申万标签**,在报告中加备注"主业分布:第一大业务 X% / 第二大 Y%"。
4. 若第一大业务占比 < 50% → **保留申万标签**,**强制加"TOC+TOB 双边"或"混合型"**子标签,Step2 用 A+B 双框架并行。

### 0.3 兜底：券商研报佐证（只在 0.1 / 0.2 矛盾时使用）

当 0.1（申万标签）与 0.2（主业归属）严重冲突（占比差距 > 30 个百分点）时：
```
westock-data report <code> --limit 5
```
取券商研报 Top 3 中"主营业务"描述的共识结论，作为最终标签，并在报告中标注"标签来源：申万分类 vs 主业占比冲突，以券商研报共识为准"。

### 0.4 §五 触发条件(由 0.1/0.2 标签决定)

| 业务模式标签 | §五 触发 |
|---|---|
| 互联网 / 平台 / 电商 / TOC SaaS / TOC+TOB 双边 / 传媒 / 计算机 / 通信 | ✅ 触发(填 §五.A 用户规模) |
| TOB SaaS / 工业品 / 制造 / 金融 / 医药 | ✅ 触发(填 §五.B 客户与留存) |
| 消费品 / 强周期 / 资源 / 地产 / 建筑 / 其他 | ❌ 不触发 |

## 六、公司所在行业分析

> **归属**：本节**仅在 profile 单步模式**使用。full 模式下行业大盘由主报告模板 §2.1 + §2.2 输出（五力 + 护城河），不在画像层重复。
> **数据原则**:用**结构化行业数据**(`westock-data sector ranking` 申万行业 + `sector constituent` 成份股 + `finance` 头部公司财务)为主;**严禁引用单一来源作总规模**。

### 6.1 行业规模

| 指标 | 数值 | 时间窗口 | 数据来源 |
|------|------|---------|---------|
| 行业总规模(亿) | | {YYYY} | |
| 过去 5 年 CAGR(%) | | {YYYY-5} ~ {YYYY} | |
| 未来 5 年 CAGR 预测(%) | | {YYYY} ~ {YYYY+5} | |
| 公司营收占行业比重(%) | | {YYYY} | |

> **解读要求**:
> - CAGR > 15% → 高增长行业,公司有 beta 红利
> - CAGR 5%-15% → 稳健增长,需看公司份额变化
> - CAGR < 5% → 成熟/低增长,**核心看公司份额提升 + 利润率扩张**
> - **公司占比** > 30% → 行业龙头,直接定位"份额维稳 + 价格维稳"
> - **公司占比** 10%-30% → 行业前列,有提份额空间
> - **公司占比** < 10% → 行业长尾,需识别差异化卡位

> **集中度解读**(参考 Bain & Company 结构-集中度矩阵):
> - **CR3 > 70%** → **寡占型**,头部协同定价能力强,新进入者极难
> - **CR3 40%-70%** → **集中型**,头部有规模优势,二线仍有空间
> - **CR3 20%-40%** → **竞争型**,格局未定,份额变化快
> - **CR3 < 20%** → **分散型**,无明显龙头,差异化或区域龙头是出路

### 6.3 行业生命周期定位

| 维度 | 当前判断 | 证据 |
|------|---------|------|
| **所处阶段** | 【导入期 / 成长前期 / 成长后期 / 成熟期 / 衰退期】 | |
| **核心驱动** | 【政策驱动 / 技术驱动 / 消费升级 / 人口红利 / 出口拉动】 | |
| **技术变革** | 【5 年内有大变革 / 10 年内有 / 稳定】 | |
| **政策友好度** | 【强政策支持 / 中性 / 强政策限制】 | |

> **生命周期与投资策略对应**:
> - **导入期**:看技术路径 + 谁能跑出来,**估值给想象空间**
> - **成长前期**:看增速 + 渗透率,**估值给高 PE**
> - **成长后期**:看份额提升 + 利润率扩张,**估值合理 PE**
> - **成熟期**:看现金流 + 分红,**估值给低 PE/PB**
> - **衰退期**:看转型 / 退出,**估值看清算价值**
