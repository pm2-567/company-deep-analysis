# 行业适配的产业链与竞争结构分析

> 本文件在 Step2 开始时加载。提供按**行业特性**适配的竞争结构分析框架。
> 加载后第一步:用"五力框架选型"确定用哪套框架,再按该框架的指标体系执行。
> **流程**:先在 Step1 做好行业分类(打业务模式标签) → Step2 §2.0 根据标签选五力框架。

## 0. 五力框架选型(必读)

> **红线规则**：业务模式标签 → 框架是**三选一互斥**关系(A 经典 / B 平台 / C 网络效应)。选了哪一框架,**全篇五力分析只用这一套**,**禁止**在主框架之外再开"补充经典 Porter 五力""平台属性速评""经典五力作为参考"等子小节。即使公司有混合业务(如腾讯有游戏+平台、美团有外卖+酒旅),也按**主业占比最高的业务模式标签**选择唯一框架,不用双框架并行。

Porter 五力(1985)是**通用框架**,但对平台/生态型公司解释力不足。Step2 开始前必须判断行业类型,选择对应框架:

```
目标公司所属赛道是什么?
│
├── 传统制造 / 重资产 / 周期型(钢铁、化工、铝、航运、地产、银行)
│   └── 用【框架 A:经典 Porter 五力】
│
├── 互联网平台 / 双边市场 / SaaS / 社交 / 内容 / 电商 / O2O
│   └── 用【框架 B:平台/生态型五力】(本文件 §2)
│       权威背书:Ben Thompson《Aggregation Theory》(Stratechery, 2014-2017)
│                 Parker, Van Alstyne & Choudary《Platform Revolution》(2016)
│                 Eisenmann, Parker & Van Alstyne《Platform Envelopment》(2011, MIT Sloan)
│
└── 网络效应型(通讯、支付、操作系统、交易所、开发工具)
    └── 用【框架 C:网络效应五力】(本文件 §3)
        权威背书:Hamilton Helmer《7 Powers》(1998/2016)
                  Metcalfe's Law + Reed's Law(网络效应量化)
```

**判定依据**(用 f10.行业分析 + f10.公司概况 字段):
- 行业关键词含"互联网/平台/电商/社交/内容/游戏/SaaS/软件/信息服务" → 框架 B
- 行业关键词含"通讯/支付/交易所/系统软件" → 框架 C
- 其他 → 框架 A

**当行业无法明确归类时**:在 Step1 公司画像阶段就要把行业归属搞清楚,不允许在 Step2 出现"无法明确归类"的情况。如果 Step1 行业字段缺失,回到 Step1 用 WebSearch 补齐"所属行业(申万/HSI/Wind 三选一)"再进入 Step2。

## 1. 框架 A:经典 Porter 五力(传统/重资产/周期型)

适用于:制造业、化工、有色金属、地产、银行、传统消费品、传统服务业。

### 分析框架

| 力量 | 对应产业链位置 | 核心问题 |
|------|-------------|---------|
| 供应商议价力 | 上游 | 供应商能否提高价格或降低质量? |
| 买方议价力 | 下游 | 客户能否压价或要求更多服务? |
| 现有竞争者威胁 | 中游(同行) | 行业内竞争有多激烈? |
| 新进入者威胁 | 潜在竞争 | 新玩家进入的难度有多大? |
| 替代品威胁 | 跨界竞争 | 有没有替代产品/服务正在侵蚀市场? |

### 逐项分析模板

#### 1. 供应商议价力(上游)

**分析要点**:
- 主要原材料/技术/服务供应方有哪些?
- 供应商集中度:前 5 大供应商采购占比(如有数据)
- 切换成本:更换供应商的成本和时间
- 是否有替代供应商
- 供应商是否有向前一体化的能力

**数据来源**:
- `f10.行业分析` / `f10.公司概况` 行业描述
- `westock-data report <代码> --limit 5`（搜索供应商 / 原材料 / 采购描述）
- `westock-data news search <行业> 产业链 上游`（行业上游供应商格局）

**评级标准**:
- **强**:供应商集中度高、切换成本高、无替代 → 目标公司议价力弱
- **中**:有一定选择空间但切换有成本
- **弱**:供应商分散、切换成本低 → 目标公司议价力强

#### 2. 买方议价力(下游)

**分析要点**:
- 主要客户群体是谁?
- 客户集中度:前 5 大客户销售占比(如有数据)
- 产品可替代性:客户能否轻易找到替代产品
- 客户价格敏感度
- 客户是否有向后一体化的能力

**数据来源**:
- `finance`(东财主要财务指标) 中的盈利能力(反映定价权)
- `westock-data report <代码> --limit 5`（客户 / 下游 / 销售描述）
- `westock-data news search <行业> 产业链 下游`（行业下游客户格局）

**评级标准**:
- **强**:客户集中度高、产品可替代、客户价格敏感 → 目标公司议价力弱
- **中**:客户有一定分散度
- **弱**:客户分散、产品不可替代、客户价格不敏感 → 目标公司议价力强

#### 3. 现有竞争者威胁

**分析要点**:
- 同行业主要竞争对手有谁?
- 市场份额分布(CR3/CR5 集中度)
- 竞争手段:价格战/差异化/服务竞争
- 行业增速:高增速行业竞争相对缓和,低增速行业竞争激烈
- 退出壁垒:产能是否容易退出

**数据来源**:
- `f10.行业分析` / `f10.公司概况` 行业地位
- 从 `/tmp/{代码}_data.json` → `em_info` 字段读取行业分类
- `westock-data sector ranking`（行业整体排名 / CR3 / 竞争格局）
- `westock-data sector constituent <板块代码>`（同行业可比公司列表）
- `westock-data report <代码> --limit 5`（竞品 / 竞争描述）

**评级标准**:
- **强**:竞争者多、份额分散、价格竞争激烈、行业低增长
- **中**:有一定竞争但格局相对稳定
- **弱**:竞争者少、份额集中、差异化竞争、行业高增长

#### 4. 新进入者威胁

**分析要点**:
- 资金壁垒:初始投资规模
- 技术壁垒:核心技术/专利保护
- 牌照壁垒:行业准入资质
- 品牌壁垒:品牌认知度和客户忠诚度
- 渠道壁垒:销售网络建设难度
- 规模壁垒:是否需要规模经济才能盈利
- 政策壁垒:环保/安全/数据等监管要求

**数据来源**:
- `f10.行业分析` / `f10.公司概况` 行业特性
- `westock-data report <代码> --limit 5`（行业准入门槛 / 牌照 / 壁垒）
- `westock-data news search <行业> 新进入者 投资 门槛`（新进入者动态）

**评级标准**:
- **强(威胁小)**:壁垒高、新进入者难以撼动
- **中**:有一定壁垒但资本雄厚者可突破
- **弱(威胁大)**:壁垒低、新玩家容易进入

#### 5. 替代品威胁

**分析要点**:
- 有哪些替代产品/服务?
- 替代品的性价比对比
- 客户转换到替代品的成本
- 替代品的发展趋势(是否在快速迭代)
- 是否有跨界竞争者(如互联网公司颠覆传统行业)

**数据来源**:
- `westock-data news search <行业> 替代品 替代方案 颠覆`
- `westock-data report <代码> --limit 5`（替代品 / 新技术 / 趋势）
- `f10.行业分析` / `f10.公司概况` 行业趋势

**评级标准**:
- **强**:替代品性价比高、客户切换成本低、替代趋势明确
- **中**:有替代品但尚未形成规模威胁
- **弱**:暂无明显替代品

## 2. 框架 B:平台/生态型五力(互联网/SaaS/双边市场)

> **方法论背书**:Ben Thompson 在 Stratechery 提出的 **Aggregation Theory**(2014-2017)指出,
> 互联网平台的核心是"聚合用户和供应商",通过控制用户关系(而非控制供给)获得超额利润。
> 平台竞争结构与传统制造业有根本不同——**用户获取成本(CAC)、留存率、LTV、网络效应、跨边网络效应**
> 这五个指标比传统"上下游议价力"更具解释力。
>
> **本框架综合**:Aggregation Theory + Parker/Van Alstyne《Platform Revolution》网络效应分类
> + 平台单位经济(unit economics)分析。

### 分析框架

| 力量 | 对应平台机制 | 核心问题 |
|------|-----------|---------|
| 用户获取成本(CAC) | 需求侧 | 平台拉新一个用户花多少钱?效率在变高还是变低? |
| 用户留存与生命周期价值(LTV) | 需求侧 | 留存多少?单个用户生命周期贡献多少收入? |
| 同边网络效应 | 同侧 | 用户越多,平台对同类型用户价值越大吗? |
| 跨边网络效应 | 双边 | 一侧用户增长能否带动另一侧增长? |
| 平台 vs 平台(替代品) | 跨平台 | 用户切换到其他平台的成本有多高? |

### 逐项分析模板

#### 1. 用户获取成本(CAC)— 替代"供应商议价力"

**分析要点**:
- 拉新成本趋势:近 3-4 期 CAC 是上升还是下降?(下降 = 平台规模效应;上升 = 流量见顶)
- 渠道结构:自然流量占比 vs 付费流量占比(自然流量占比高 = 平台心智强)
- 销售费用率:销售费用 / 营收(行业前 10% vs 中位数对比)
- 头部效应:头部 1% 用户贡献多少?长尾多少?(衡量依赖度)

**数据来源**:
- 利润表销售费用、管理费用、研发费用
- `westock-data report <代码> --limit 5`（用户获取 / 拉新成本 / 留存）
- `westock-data news article <代码> --limit 10`（MAU / DAU / 付费用户 / 财报披露）
- 招股书 / 年报"经营情况讨论与分析"章节

**评级标准**:
- **强(护城河深)**:CAC 持续下降 + 自然流量占比 > 60% + 销售费用率低于行业中位数
- **中**:CAC 趋稳 + 销售费用率行业中位数附近
- **弱(护城河浅)**:CAC 上升 + 付费流量依赖 + 销售费用率高于行业中位数

#### 2. 用户留存与 LTV — 替代"买方议价力"

**分析要点**:
- 次日/7 日/30 日留存率(招股书或券商深度报告常披露)
- 12 个月留存率(> 40% 算优秀,< 20% 警惕)
- 流失率趋势(Churn Rate):LTM 流失率 vs 上年同期
- LTV / CAC 比值:> 3 健康,1-3 警戒,< 1 不可持续
- 付费转化率:免费用户 → 付费用户的转化漏斗

**数据来源**:
- 招股书"业务与技术" / "经营情况"章节
- `westock-data report <代码> --limit 5`（留存率 / 流失率 / LTV）
- 财报"用户数据"披露(互联网公司常规披露)

**评级标准**:
- **强**:12 月留存 > 40% + LTV/CAC > 3 + 流失率下行
- **中**:12 月留存 20%-40% + LTV/CAC 1-3
- **弱**:12 月留存 < 20% + LTV/CAC < 1

#### 3. 同边网络效应 — 替代"现有竞争者威胁"

**分析要点**:
- 用户规模越大,平台对同类型用户价值是否增加?
  - 例:微信用户越多 → 通信价值越大(强同边)
  - 例:LinkedIn 用户越多 → 求职价值越大(中等)
  - 例:Netflix 用户越多 → 订阅价值不变(无)
- 头部 KOL / 商家集中度:CR10 商家占 GMV 多少?
- 平台 MAU/DAU 比值(粘性):> 30% 高粘性,< 15% 工具型
- 用户日均使用时长趋势(YoY)

**数据来源**:
- 招股书 / 年报"用户数据"章节
- `westock-data news article <代码> --limit 10`（MAU / DAU / 黏性 / 使用时长）

**评级标准**:
- **强**:同边网络效应明显 + MAU/DAU > 30% + 使用时长持续增长
- **中**:有一定粘性但增速放缓
- **弱**:工具型平台,使用即走

#### 4. 跨边网络效应 — 替代"新进入者威胁"

**分析要点**:
- 需求侧用户增长是否能带动供给侧(商户/开发者/创作者)增长?
  - 例:淘宝买家越多 → 商家越多(强跨边)
  - 例:滴滴乘客越多 → 司机越多(强跨边)
  - 例:微信用户越多 → 公众号创作者越多(中等)
- 供给侧集中度:头部供给方占比 + 议价力
- 平台对供给侧是否补贴?(补贴退坡后是否会流失?)
- 飞轮效应是否已经转起来?

**数据来源**:
- 招股书 / 年报"双边市场" / "网络效应"披露
- `westock-data report <代码> --limit 5`（双边市场 / 飞轮 / 网络效应）
- `westock-data news search <行业> 多归属 跨边网络 平台经济学`（行业平台经济学）

**评级标准**:
- **强(壁垒深)**:跨边网络效应已验证 + 头部供给方难离开 + 飞轮稳定
- **中**:跨边效应存在但较弱
- **弱**:单边平台,跨边效应不明显

#### 5. 平台 vs 平台(替代品)— 替代"替代品威胁"

**分析要点**:
- 用户多归属(multi-homing)成本:用户在多个平台同时存在吗?
  - 社交平台:低多归属(微信一家独大)
  - 内容平台:高多归属(抖音 + B站 + 小红书并行)
  - 电商平台:中等(用户多个 App 都装)
- 切换成本:数据迁移、关注关系重建、商家资质重新审核
- 平台可替代性:若主要竞品是同类型平台(微信 vs QQ),还是跨类型(微信 vs 钉钉)?

**数据来源**:
- `westock-data report <代码> --limit 5`（用户多归属 / 切换成本 / 平台竞争）
- `westock-data news search <行业> 头部平台 市场份额 跨平台`（行业头部平台格局）

**评级标准**:
- **强(护城河深)**:用户低多归属 + 切换成本高 + 竞品为同类型有限几家
- **中**:多归属存在但平台差异化明显
- **弱(护城河浅)**:用户高多归属 + 切换成本低 + 跨类型竞品多

### 输出格式(框架 B 专用,合并到模板 §2.2 通用 5 行)

| 力量(框架 B 视角) | 评级(强/中/弱) | 对公司影响 | 关键论据(网络效应 / 单位经济 / 平台 vs 平台) |
|---|---|---|---|
| 供应商议价力 → 平台供给侧 | | | KOL/商家集中度、补贴依赖度 |
| 买方议价力 → 用户侧 | | | 用户多归属率、补贴敏感度、切换成本 |
| 现有竞争者 → 平台 vs 平台 | | | 同类平台数、差异化程度、LTV/CAC |
| 新进入者威胁 → 跨类型竞品 | | | 跨类型(自营/社交/工具)替代压力 |
| 替代品威胁 → 飞轮/单边增速 | | | 同边 + 跨边网络效应强度、单位经济回收期 |

**单位经济(unit economics)速判**:LTV/CAC = __,回收期 = __ 月

**护城河类型**:【规模效应型 / 网络效应型 / 数据资产型 / 生态绑定型】
```

## 3. 框架 C:网络效应五力(交易所/支付/通讯/系统软件)

> **方法论背书**:Hamilton Helmer 在《7 Powers》(1998/2016)中系统化定义了 **7 种竞争优势来源**,
> 其中与网络效应最相关的有 4 种:网络经济(Network Economy)、规模经济(Economy of Scale)、
> 转换成本(Switching Cost)、品牌(Brand)。本框架以这 4 种 Power 为核心,辅以 Metcalfe/Reed 定律做量化。

### 适用场景
- 交易所(沪深交易所、港交所、币安)
- 支付网络(银联、支付宝、Visa、PayPal)
- 操作系统(Windows、iOS、Android)
- 通讯网络(中国移动、运营商)
- 开发工具(GitHub、AWS、VS Code)
- 大型多人游戏(LOL、王者荣耀)

### 分析框架

| 维度 | 核心问题 | 量化指标 |
|------|---------|---------|
| **网络经济**(同边 + 跨边) | 用户/节点数翻倍,价值是否翻倍以上? | Metcalfe 定律 V = n² ;Reed 定律 V = 2ⁿ |
| **节点分布与马太效应** | 头部节点占多少流量?是否赢家通吃? | CR10 节点流量占比、基尼系数 |
| **多归属与切换成本** | 用户能否低成本切换到其他网络? | 多归属比例、协议标准锁定、API 锁定 |
| **生态厚度** | 网络上跑了多少互补品/应用? | 第三方应用数、API 调用量、SDK 下载量 |
| **技术演进威胁** | 是否有新一代协议/标准正在替代? | Web3 替代 Web2、IPv6 替代 IPv4 |

### 评级标准(框架 C)

- **强护城河**:网络效应已被验证 + 头部节点高度集中 + 切换成本高 + 生态厚度持续扩大
- **中**:有网络效应但被新进入者部分削弱
- **弱**:网络效应尚在建立中 / 面临技术替代
