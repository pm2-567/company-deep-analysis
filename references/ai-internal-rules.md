## §0 元信息规则

### 0.1 基准年判定（重要，写报告必须按此判定）

`{基准年}` = 当前时间最近一个已披露完整年报的年份

| 当前时间 | 基准年 | 例子 |
|---------|--------|------|
| 4 月 1 日之前 | 前一年年报 | 2026-02 跑 → 基准年=2025 |
| 4 月 1 日及之后（年报已披露） | 当年年报 | 2026-05 跑 → 基准年=2026 |

- `{基准年-1}` = 基准年的前一年
- `{基准年-1}→{基准年} 营收 CAGR` = 同比增速（跨度=1 年时）
- 报告"摘要"必须说明选用的基准年（模板 §6 元信息表里有 `基准年` 列，照填）

### 0.2 数据质量等级 A/B/C

| 等级 | 判定标准 |
|------|---------|
| **A** | 核心指标全部有数据，CLI 直采 + 多源交叉验证 |
| **B** | 核心指标 2-3 期有数据，1 个来源 |
| **C** | 关键数据缺失，需 WebSearch 补充估算 |

### 1.1 用户规模数据原则（互联网/平台/TOB SaaS 触发）

- **优先**：年报/招股书/季报官方披露

### 1.2 摘要 3 句话设计原则

- **第一句**：动作 + 标的 + 价位 + 空间 + 时间（5 要素齐全）

- **第三句**：风险 + 触发条件（不只说"风险"而说"什么条件下会怎样"）
- **三句话内不出现公式、不列数字明细**

## §3 财务分析规则

### 3.0 财务权重（按业务模式选唯一一套）

详见 `references/financial-analysis-guide.md` §0.1。**7 套行业调整表，6 选 1 互斥**（"其他"无独立类别，回 Step1 重判）。

### 3.1-§3.4 指标分析写法硬要求

- **每条分点必须"结论先行"**——先给判断（"毛利率高于行业 10pct"），后接数据/事实（"55% vs 行业中位数 45%"）
- **主语**：公司名（{公司名} / 公司 / 业务）
- **谓语**：具体动词（高于/低于/接近/超过/显著领先）
- **宾语**：具体数据/对比对象
- **结构**：① 一句话总论（强/中/弱 + 关键定性特征）→ ② 3-5 个分点（结论先行 + 数据/事实）

### 3.5 雷达图

- 双层雷达：外层=实际得分 0-100，内层=行业中位数
- 实现：`matplotlib` + macOS 自带中文字体（`Heiti TC` / `Songti SC`）
- 嵌入：PNG 读取为 base64 → 内嵌到 markdown `![描述](data:image/png;base64,...)`

### 3.6 杜邦分解（所有行业必做）

- 公式：ROE = 净利率 × 总资产周转率 × 权益乘数
- 亏损公司用替代公式：ROA + 扣非 ROE（详见 `financial-analysis-guide.md` §1.A）

## §6 可比公司规则

### 6.1 数据获取（强制 CLI 直采）

```bash
python3 scripts/collect_data.py --finances <目标公司代码>,<可比1>,<可比2>,<可比3>
```

- 一次拉全所有公司的最近一期快照（毛利率/净利率/ROE/营收/同比/资产负债率/8 期报告期列表）
- 精度 = 东财数据中心原始数据
- **可比公司财务比率走 CLI 直采**（WebSearch 估算精度不够）

### 6.2 列名映射（`get_finance()` 返回 → 表格列）

| get_finance 字段 | 表格列 |
|-----------------|--------|
| 营业总收入 | {基准年} 营收（亿） |
| 销售毛利率 | {基准年} 毛利率 |
| 销售净利率 | {基准年} 净利率 |
| ROE加权 | {基准年} ROE |
| `_近8期报告期` | 选最近一个已披露完整年报的报告期作为"基准年"那一行 |

### 6.3 可比性等级

- 🟢 **高可比**：业务模式最相近
- 🟡 **中可比**：业务模式部分重叠
- 🔴 **低可比**：仅作产业链/规模参考

## §7 估值规则

### 7.1 估值方法（按行业选 1-3 种，必须如实标注）

- **通用**：PE-TTM / PB-MRQ / EV/EBITDA-TTM
- **亏损/早期**：PS-TTM / DCF（远期）
- **金融**：PB + 股息率 + 不良率
- **地产**：NAV 重估
- **港股**：PE-Forward 港股无一致预期数据，**跳过**
- **D&A 反推算法**（OCF > 净利时有效）：`D&A = OCF 净额 - 净利润`
- **反虚标**：只列"实做"方法，未做不写

### 7.2 估值结论

- 综合得分（按权重 35/10/5/30 之类）必须列加权公式
- 目标价给出区间 + 对应方法 + 数据来源
